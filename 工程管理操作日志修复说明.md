# 工程管理操作日志路径修复

## 问题描述
工程管理模块的"操作日志"菜单配置错误，`component` 字段指向了 `monitor/operlog/index`，但实际应该指向 `project/operlog/index`。

## 问题原因
数据库配置部分内容位于 `ruoyi-fastapi-backend/sql/project.sql` 中。

## 修复内容

### 1. 数据库配置修复
修改文件：`ruoyi-fastapi-backend/sql/project.sql`

**修改前：**
```sql
VALUES(119, '操作日志', 4, 2, 'operlog', 'monitor/operlog/index', 'projectOperLog', ...)
```

**修改后：**
```sql
VALUES(119, '操作日志', 4, 2, 'operlog', 'project/operlog/index', 'projectOperLog', ...)
```

### 2. 前端页面说明
前端页面 `ruoyi-fastapi-frontend/src/views/project/operlog/index.vue` 已经正确实现：
- 使用 `OperlogGrid` 组件
- 通过 `:filter-module="'工程管理'"` 过滤只显示工程管理相关的操作日志
- 保持与监控模块操作日志相同的功能（删除、清空、导出）

## 应用修复

### 方式一：重新执行 SQL 脚本（推荐）
```bash
mysql -u root -p ruoyi-fastapi < ruoyi-fastapi-backend/sql/project.sql
```

### 方式二：手动更新数据库
```sql
UPDATE sys_menu 
SET component = 'project/operlog/index' 
WHERE menu_id = 119 AND menu_name = '操作日志' AND parent_id = 4;
```

## 验证
1. 重启前端应用
2. 登录系统
3. 进入"工程管理" -> "操作日志"
4. 确认页面正确加载，且只显示工程管理相关的操作记录
