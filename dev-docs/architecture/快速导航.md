# 架构文档快速导航

## 📚 文档概览

本架构文档为 RuoYi-Vue3-FastAPI 项目提供完整的技术架构说明,帮助新开发者快速了解项目结构并开始开发。

## 🎯 学习路径

### 新手入门 (第1天)
1. 阅读 [README](./README.md) 了解文档结构
2. 查看 [后端系统概览](./backend/01-系统概览.md) 了解后端技术栈
3. 查看 [前端系统概览](./frontend/01-系统概览.md) 了解前端技术栈
4. 运行项目,熟悉基本功能

### 后端开发者 (第2-3天)
1. [分层架构](./backend/02-分层架构.md) - 理解 Controller/Service/DAO 模式
2. [模块结构](./backend/03-模块结构.md) - 了解功能模块划分
3. [数据库设计](./backend/04-数据库设计.md) - 熟悉数据表结构
4. [API设计](./backend/05-API设计.md) - 学习 API 开发规范
5. [权限认证](./backend/06-权限认证.md) - 掌握权限系统
6. [开发规范](./backend/07-开发规范.md) - 遵循编码标准

### 前端开发者 (第2-3天)
1. [组件架构](./frontend/02-组件架构.md) - 学习 Vue 3 组件开发
2. [状态管理](./frontend/03-状态管理.md) - 掌握 Pinia 使用
3. [API集成](./frontend/05-API集成.md) - 了解 API 调用方式
4. [权限系统](./frontend/06-权限系统.md) - 理解前端权限控制
5. [开发规范](./frontend/07-开发规范.md) - 遵循编码标准

### 全栈开发者 (第4-5天)
1. 完整阅读后端和前端所有文档
2. 理解前后端交互流程
3. 掌握完整的开发流程
4. 开始实际开发任务

## 💡 常见问题

### Q: 我是新手,应该从哪里开始?
A: 先阅读两个系统概览文档,然后根据你的角色(后端/前端)选择对应的学习路径。

### Q: 文档太多,我应该全部看完吗?
A: 不需要。根据你的开发任务,选择相关的文档阅读即可。建议先看概览和架构,再看具体的开发规范。

### Q: 我想快速上手开发,最少要看哪些?
A: 
- 后端: 系统概览 → 分层架构 → 开发规范
- 前端: 系统概览 → 组件架构 → 开发规范

### Q: 文档和实际代码不一致怎么办?
A: 以实际代码为准,同时欢迎提出文档更新建议。

## 🚀 开始开发

### 环境准备
1. 安装 Python 3.9+
2. 安装 Node.js 16+
3. 安装 MySQL 5.7+ 或 PostgreSQL
4. 安装 Redis

### 快速启动
```bash
# 首次运行
./setup.sh

# 日常开发
./start.sh
```

### 开发流程
1. 创建功能分支
2. 编写代码(遵循开发规范)
3. 本地测试
4. 提交代码(遵循 Git 规范)
5. 发起 Pull Request
6. 代码审查
7. 合并到主分支
