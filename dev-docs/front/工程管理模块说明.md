# 工程管理模块

## 功能说明

工程管理模块是一个简化的CRUD模块，包含以下字段：
- **工程编号**：自动生成的主键ID
- **工程描述**：工程的文字描述
- **创建人**：创建该工程记录的用户
- **创建时间**：记录创建时间
- **更新时间**：记录最后更新时间

## 部署步骤

### 1. 数据库初始化

执行SQL脚本创建表和菜单：

```bash
mysql -u root -p ruoyi-fastapi < ruoyi-fastapi-backend/sql/project.sql
```

或者手动在数据库中执行 `ruoyi-fastapi-backend/sql/project.sql` 文件。

### 2. 后端部署

后端代码已经自动集成到项目中，无需额外配置。重启后端服务即可：

```bash
cd ruoyi-fastapi-backend
python app.py --env=dev
```

### 3. 前端部署

前端代码已创建，但需要在路由中注册。前端会自动从后端获取菜单配置。

### 4. 权限配置

使用管理员账号登录系统后：

1. 进入 **系统管理 → 菜单管理**
2. 找到"工程管理"菜单（如果SQL执行成功会自动创建）
3. 进入 **系统管理 → 角色管理**
4. 为需要的角色分配"工程管理"相关权限

## 功能特性

### 后端实现

- **Controller**: `ruoyi-fastapi-backend/module_admin/controller/project_controller.py`
- **Service**: `ruoyi-fastapi-backend/module_admin/service/project_service.py`
- **DAO**: `ruoyi-fastapi-backend/module_admin/dao/project_dao.py`
- **DO模型**: `ruoyi-fastapi-backend/module_admin/entity/do/project_do.py`
- **VO模型**: `ruoyi-fastapi-backend/module_admin/entity/vo/project_vo.py`

### 前端实现

- **页面**: `ruoyi-fastapi-frontend/src/views/project/index.vue`
- **API**: `ruoyi-fastapi-frontend/src/api/project/project.js`

### API接口

- `GET /project/list` - 获取工程列表（分页）
- `GET /project/{id}` - 获取工程详情
- `POST /project` - 新增工程
- `PUT /project` - 修改工程
- `DELETE /project/{ids}` - 删除工程

### 权限标识

- `project:project:list` - 工程列表查询
- `project:project:query` - 工程详情查询
- `project:project:add` - 工程新增
- `project:project:edit` - 工程修改
- `project:project:remove` - 工程删除

## 页面功能

### 列表页面

- 支持按创建时间范围搜索
- 支持分页显示
- 支持批量删除
- 显示字段：工程编号、工程描述、创建人、创建时间、更新时间
- 操作按钮：详情、修改、删除

### 详情页面

以对话框形式展示工程的完整信息。

### 新增/编辑页面

以对话框形式提供工程描述的输入。

## 测试访问

1. 启动后端服务：`http://localhost:8000`
2. 启动前端服务：`http://localhost:80`
3. 使用管理员账号登录（admin/admin123）
4. 在左侧菜单找到"工程管理"
5. 进行增删改查操作测试

## API文档

启动后端后，可以访问自动生成的API文档：
- Swagger UI: `http://localhost:8000/docs`
- 在文档中搜索"工程管理"查看所有接口

## 注意事项

1. 所有接口都需要登录认证
2. 所有操作都需要相应的权限
3. 删除操作为逻辑删除，不会真正删除数据
4. 创建人和时间由系统自动填充
5. 工程描述字段为可选字段，可以为空
